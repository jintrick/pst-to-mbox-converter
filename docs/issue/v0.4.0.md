# Issue v0.4.0: アドレス帳エクスポート機能の実装

## 前提知識
- `docs/issue/v0.3.0.md`: `ctypes` を用いた `libpff` の直接バインディングへの移行経緯。
- `docs/issue/v0.3.1.md`: `libpff` ライブラリの同梱と動的ロードに関する現在のアーキテクチャ。
- `docs/reference/libpff.3`: `libpff` のAPIリファレンス。

## 現状の問題点

現在のツールはPSTファイルから**メール**を抽出しMBOX形式で保存することに特化している。
アドレス帳（連絡先）や予定表など、メール以外のアイテムをエクスポートする機能が存在しないため、PSTファイルからの完全なデータ移行ができない。

## 解決策の概要

PSTファイル内のアドレス帳（連絡先）を抽出し、Thunderbirdなどの主要なメールクライアントがインポート可能な**CSV形式**で出力する新機能を実装する。

この機能は、新しいコマンドライン引数 `--type contacts` を指定することで有効になる。

## 解決策の詳細

### 1. コマンドラインインターフェースの拡張

`main.py` を改修し、以下の引数を追加・変更する。

- `--type`: エクスポートするアイテムの種類を指定する。
  - `mail` (デフォルト): 従来のメールエクスポート機能。
  - `contacts`: 新しく実装するアドレス帳エクスポート機能。
- `--output`: 出力ファイルのパス。
  - `--type contacts` の場合、このパスは `.csv` ファイルを指すことが期待される。

**実行コマンド例:**
```bash
pst-converter --input /path/to/file.pst --output /path/to/contacts.csv --type contacts
```

### 2. `pst_reader.py` の機能拡張

`libpff` を利用して、メールだけでなく連絡先アイテムも読み取れるように `PSTReader` クラスを拡張する。

- **アイテムタイプの判別**:
  - `libpff` の `libpff_item_get_type` 関数や、メッセージクラス (`IPM.Contact` など) を示すプロパティを読み取ることで、アイテムが連絡先であるかを判別するロジックを追加する。
- **連絡先データの抽出**:
  - 連絡先アイテムから、名前、メールアドレス、電話番号などの情報を抽出する。`libpff_record_set_get_entry_by_type` や `libpff_record_entry_get_data_as_utf8_string` などの関数を利用して、MAPIプロパティタグ（例: `0x3001` - 表示名）に対応する値を取得する。
- **`get_contacts()` ジェネレータの実装**:
  - `get_messages()` と同様に、連絡先データを辞書オブジェクトのジェネレータとしてyieldする `get_contacts()` メソッドを新設する。

### 3. `csv_writer.py` の新規作成

抽出した連絡先データをCSVファイルに書き込む責務を持つ `csv_writer.py` と `CSVWriter` クラスを `src/pst_to_mbox_converter/` ディレクトリに新規作成する。

- **`__init__`**: 出力ファイルのパスを受け取り、CSVヘッダー（例: `Display Name`, `Email Address`, `Business Phone`）を書き込む。
- **`add_contact()`**: `pst_reader` から受け取った連絡先データ（辞書）をCSVの1行として書き込むメソッドを実装する。

- メールアドレスの取得は複雑になる可能性があります。PSTの仕様では複数のメールアドレスフィールドが存在するため、フォールバックロジックを実装する必要があります。

## テスト方法
- テストコードは、`test/`ディレクトリに配置してください。
- 連絡先を含むテスト用のPSTファイル (`test/data/contacts.pst` など) を新規に用意する必要があります。
- `main` のテストでは、`--type contacts` を指定して実行し、生成されたCSVファイルの内容が期待通りであることを検証します。
